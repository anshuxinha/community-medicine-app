<task type="auto">
  <name>Task 1: Implement Transactional Coupon Increments</name>
  <files>src/services/couponService.js</files>
  <action>
    Refactor `incrementCouponUsage` to use Firestore's `runTransaction`. 
    Inside the transaction:
    1. Read the coupon document using the transaction.
    2. Check if the coupon exists and if `usageLimit` is defined.
    3. If `usageCount >= usageLimit`, throw an error (or log and skip increment to avoid over-subscription).
    4. Update `usageCount` by `(currentCount || 0) + 1`.
    
    This ensures that concurrent purchases don't bypass the usage limit (per Research Pitfall 2).
  </action>
  <verify>
    <automated>grep -q "runTransaction" src/services/couponService.js</automated>
  </verify>
  <done>incrementCouponUsage is atomic and enforces usage limits during the write phase.</done>
</task>
